System Requirements

Functional Requirements

Basic Cruise Control

  • The car and cruise system must be turned on.
  • The system will maintain the user-defined speed within ±1 mph on grades ±5%.
  • The cruise control system temporarily reverts to manual control if the brake is pressed.
  • If the accelerator pedal is pressed while cruise control is active, the system will allow temporary speed increase as long as the pedal is pressed, but resume back to the set speed when released.

Detection and Distance Control

  • Maintain a continuously updated in-path lead target.
  • Update target range and range-rate in an adequate amount of time.
  • Allow user to select a time headway setting: short (~0.8s), medium (1.2s), long (1.8s).
  • Maintain gap within ±0.2s.
  • If the lead decelerates, the system will decelerate to a comfortable limit. In emergencies, the limit will be removed.
  • System will stop and hold brake pressure if lead vehicle is stopped.
  • System will resume movement if the lead vehicle starts moving.

Driver Interface

  • The driver can adjust system settings, such as following distance or speed.

Fallback & Fault Handling

  • If the sensor is blocked or confidence is degraded while cruise control is active, the system will sound an alarm for the user to take over.
  • If the sensor or confidence is degraded before cruise control is active, the system will not engage.
  • On critical actuator or network faults, the system will provide zero torque or brake and disengage with audible and visual alerts.

Non-functional Requirements

  • Stop Time/Distance: The system must respond within acceptable time and distance thresholds.
  • Security: The system must be protected against unauthorized access or tampering.
  • Diagnostics: The system presents a warning message if any components, such as sensors or speed adjustment software, stop working.
